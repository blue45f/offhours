# Offhours — Product Requirements

> "비어 있던 그 시간, 가장 멋진 공간이 됩니다."
>
> 카페·레스토랑·호프집 등 상업 시설의 **영업 외 시간**(휴무일, 마감 이후)만 전문으로
> 쪼개서 파티·스몰웨딩·모임·팝업 공간으로 매칭하는 양면시장(C2C) 플랫폼.

## 1. 문제 정의

- **호스트(상업 시설 사장)**: 부동산 임대료는 24시간 발생하지만 매출은 영업시간에만 발생.
  주 40시간 이상이 데드 캐피털.
- **게스트(모임 주최자)**: 파티룸·웨딩홀은 비싸고 천편일률. 카페·바 통째로 빌리는 감성
  공간은 검색·예약·결제·정산이 비표준화돼 카톡 문의로 진입 장벽이 높음.
- **기존 플랫폼의 한계**: 스페이스클라우드·아워플레이스는 "전업 호스트"에 최적화되어
  부업형 호스트가 등록·운영하기 무거움. 영업 외 시간 자동 슬롯·청소 SLA·BYOB 정책 같은
  **영업 외 시간 전용 요구**가 빠져 있음.

## 2. 핵심 가설

| #   | 가설                                                                      | 검증 신호                   |
| --- | ------------------------------------------------------------------------- | --------------------------- |
| H1  | 호스트는 "0클릭에 가까운 등록"이면 부업으로 등록한다                      | 등록 완료까지 평균 ≤ 10분   |
| H2  | 게스트는 "통대관 무드 사진 + 영업 외 시간 자동 슬롯"에 더 빠르게 결제한다 | 상세 → 결제 전환율 ≥ 12%    |
| H3  | 청소·원상복구 자동 대행이 호스트 가장 큰 진입 장벽이다                    | 청소 옵션 attach rate ≥ 60% |
| H4  | "조용한 럭셔리" 무드는 SNS 공유율에서 정량적으로 우위                     | 공유 CTR ≥ 8%               |

## 3. 타겟 페르소나

### 3.1 호스트 — "정민, 35, 망원동 카페 사장"

- 영업시간 10~22시. 월요일 휴무. 마감 후 22~02시는 데드 타임.
- 등록 시 가장 큰 걱정: ①원상복구·기물 파손, ②다음 영업 준비 시간 침범, ③주류 책임.
- 동기: 월 50~150만 원 부가 수익, 공간 노출로 본업 카페 홍보 효과.

### 3.2 게스트 A — "지은, 29, 스몰웨딩 신부"

- 예식 비용 부담으로 50~80인 규모 카페 통대관 + 자체 케이터링 희망.
- 의사결정: 무드(72%) > 가격(18%) > 위치(10%).
- 결제 의사: 시간당 5~12만 원, 4~6시간 패키지.

### 3.3 게스트 B — "현우, 31, 회사 송년회 총무"

- 30명 부서 송년회, 1차 식사 후 2차 파티 공간 필요.
- 의사결정: 위치(접근성) > 가격 > 무드.
- 결제 의사: 야간 4시간 60~100만 원, 법인카드 즉시 결제.

### 3.4 게스트 C — "유진, 27, 브랜드 마케터"

- 신제품 팝업·인플루언서 클래스 공간 탐색.
- 의사결정: 무드 + 촬영 가능 여부 + 채광/층고.
- 결제 의사: 평일 종일 200~500만 원, 세금계산서 필수.

## 4. 차별화 포인트 (USP)

> 스페이스클라우드 · 아워플레이스 · Peerspace · Tagvenue 대비 **"시간 × 위치
> × 즉시성"** 을 핵심 무기로 삼는다. 정보 나열식이 아닌, **지금 · 가까이 ·
> 비어있는** 공간을 우선 매칭하는 _라스트미닛 퍼스트_ 큐레이션.

### 4.1 시간 차별점 — Live Off-hours

1. **영업 외 시간 자동 슬롯** — 호스트가 영업시간만 입력하면 휴무일/마감 후 슬롯을 자동
   생성. "오늘 23시부터 빌릴 수 있어요" 같은 라스트미닛 노출.
2. **"지금 비어있는" 라이브 매칭** — 현재 시각 기준 N시간 내 시작 가능한 슬롯이 있는 공간을
   우선 노출. 카드에 "오늘 22:00부터 4시간 가능" 라이브 뱃지. 검색 필터 한 클릭.
3. **청소·원상복구 SLA + 체크아웃 체크리스트** _(부분 구현, 사이클6)_ — 예약 종료 후
   1~3시간 청소 윈도우 강제 포함. 호스트 체크아웃 시 **원상복구·쓰레기·음향·조명·잠금
   5개 필수 항목**을 모두 확인해야 COMPLETED 처리. 게스트는 카톡으로 받은 6자리 코드 옆에
   QR 이미지가 함께 떠 호스트가 1초 만에 체크인 가능. 제휴 청소업체 자동 매칭은 Phase 2.
4. **동적 가격 + 라스트미닛 할인** _(구현됨, 사이클3)_ — 휴무일 종일가 < 영업종료 후 야간가
   < 주말 새벽 프리미엄. **시작 6h 내 5%, 3h 내 10%, 1h 내 15%** 자동 차감이 견적·결제·
   카드 뱃지에 동시 적용. 호스트는 비어 가는 시간을 채우고, 게스트는 같은 공간을 더 싸게
   잡는 win-win 동학.
5. **영업외 슬롯 캘린더** _(구현됨, 사이클24)_ — 상세 예약 패널에서 향후 6주를 한눈에
   보여주고, 열린 영업 외 슬롯이 있는 날만 강조한다. 날짜를 누르면 그날의 슬롯 칩(예:
   `23:00–06:00 · 7시간`)이 뜨고, 칩을 누르면 **자정을 넘기는 야간 블록까지** 시작·종료가
   자동 입력돼 견적이 즉시 계산된다. 일반 예약 캘린더가 "영업 중 빈 시간"을 보여주는 것과
   달리, 마감 후·휴무일에 자동 생성된 슬롯만 노출해 "영업 외 자동 슬롯" USP를 게스트가 직접
   만지는 표면으로 만든다.

### 4.2 위치 차별점 — Hyper-local

5. **반경 기반 검색 + 거리순 정렬** — 브라우저 Geolocation → "내 위치에서 1·3·5·10km"
   토글. 카드에 "0.6km" 거리 칩 노출. 헤더 상단에 "내 위치로 보기" 핀.
6. **'오늘 인근' 홈 섹션** — 비로그인도 권한만 주면 동·구 단위로 "오늘 망원동에서 비어있는
   5개" 같은 마이크로-큐레이션 모듈 노출.
7. **동선 추천(번들)** _(구현됨, 사이클2)_ — 상세 페이지 "이 동네 동선 후보" 섹션이 1km
   내 다른 카테고리 공간을 우선 노출. "1차 다이닝 → 2차 통대관" 시나리오를 한 화면에서
   설계 가능. 스페이스클라우드·아워플레이스·Peerspace에는 없는 차별점.
   1-탭 다중 예약은 Phase 3.

### 4.3 신뢰 차별점 — Operational Trust

8. **호스트 응답 속도 뱃지** _(구현됨, 사이클1)_ — "보통 1·3·12·24시간 안에 답해요" 라벨이
   검색 카드와 상세 호스트 섹션에 동시 노출. Airbnb / Peerspace / Tagvenue 합성 룰로
   **최근 30일 신규 문의 ≥10건 + 24h 이내 응답률 ≥90%** 동시 만족 시에만 표시(통계 왜곡
   방지). 중앙값은 EMA(α=0.3) 캐시로 예약 승인/거절 시 즉시 갱신.
9. **호스트 신뢰점수 자동 갱신 + 게이지** _(구현됨, 사이클4)_ — 평점(5점 +3 ~ 1점 -5)
   · 호스트 거절 -2 · 게스트 단기 취소 -2 · 분쟁 -15 이벤트가 0~100 클램프된
   `trustScore` 에 실시간 반영. 상세 페이지에 가로 게이지 + **최고 / 우수 / 양호 / 보통 /
   주의** 5단 티어 라벨(주의 = amber 위험 신호). Airbnb Superhost / Peerspace Power Host
   / Tagvenue Supervenue 의 단일 라벨 한계를 다단계 + 동적 산정으로 넘어선다.
10. **주류·식자재 정책 템플릿** — 일반음식점/휴게음식점/식품판매업 유형별 BYOB·반입주류·
    외부 케이터링 룰을 토글 선택, 게스트 페이지에 자동 표기. 식품위생법 가이드.
11. **"법적 안전 카테고리" 우선 노출** — `SAFE_CATEGORIES` (FITNESS / DANCE / PRACTICE /
    WORKSHOP / MEETING / STUDIO / GALLERY) 호스트 온보딩 시 우선 안내.

### 4.4 의사결정 차별점

12. **공간 비교 보드 + 공유 링크** _(구현됨, 사이클5)_ — 카드 우측 GitCompare 아이콘으로
    최대 4개 공간 담아 `/compare` 5열 테이블에서 한눈에 비교. 가격·인원·신뢰·응답·즉시예약·
    주류 룰 등 12개 항목, 행마다 "최적" 값에 자동 강조. URL `?spaces=a,b,c` 한 줄로
    카톡 공유 가능. 스몰웨딩 신부·회식 총무가 모임원과 의사결정을 좁히는 핵심 마찰을 해소.

13. **모임 유형(Use Case) 기반 탐색** _(구현됨, 사이클8)_ — 카테고리(공급자 시각)는 두고,
    게스트가 실제로 검색하는 멘탈모델 12종을 별도 분류로 추가: 친구 생일파티 · 스몰웨딩 ·
    팀빌딩/송년회 · 사내 워크샵 · 베이비샤워 · 동호회 모임 · 원데이클래스 · 세미나/네트워킹 ·
    프로필/스냅 촬영 · 영상/광고 촬영 · 팝업·전시 · 리허설/연습. 홈에 "어떤 모임이세요?" 6열
    그리드(이모지+힌트)로 진입, 검색에 multi-select 칩 바, 디테일에 "이런 모임에 좋아요"
    클릭형 칩으로 cross-link. 같은 카페가 BIRTHDAY/BABYSHOWER/CLASS 여러 큐에 노출되어
    long-tail SEO 와 게스트 발견율을 동시에 끌어올린다. Peerspace 의 Event Type 모델을
    한국화한 것으로, 스페이스클라우드/아워플레이스에는 부재.

14. **컬렉션 (Wishlists) + 공유 가능 보드 + 협업 투표** _(구현됨, 사이클9·사이클13)_ —
    비교 보드(4개 단발)와 보완 관계의 영구 저장소. 게스트가 "26살 여름 결혼식 후보",
    "3분기 사내 워크샵 후보", "친구 깜짝 생일 (비공개)" 처럼 후보 공간을 폴더로 정리.
    공개 컬렉션은 슬러그 URL `/c/summer-wedding-2026` 한 줄로 친구·동료에게 공유.
    **(사이클13 협업)** 친구가 슬러그 URL로 들어와 닉네임 한 번만 입력하면 각 후보에
    👍/👎 1인 1표 (voterToken localStorage 기반). 헤더에 전체 tally + "1위" 후보 자동
    표시, 각 카드에 투표 카운트 + 투표자 목록("지원, 민호, 서연 외 2명") 노출.
    Pinterest Collab Board 의 공간대여 버전 — 그룹 의사결정 친화. Airbnb/스페이스
    클라우드/아워플레이스 모두 ✕.

15. **1/N 분담 결제 (Split Payment)** _(구현됨, 사이클11)_ — 한국 모임 문화 fit: 1명이
    공간을 한 번에 결제한 뒤 N명에게 청구 링크(`/pay/:token`)를 카톡으로 공유, 각자
    본인 부담분만 토스로 송금. 예약 디테일에 "3/8 명 정산 완료" 진척 보드 + 멤버별
    링크 복사/공유 버튼 + "미송금 멤버 링크 전체 복사" 일괄 공유. 청구 페이지는 토큰
    인증만 필요해서 로그인 없이 친구가 바로 송금. 토스 송금은 mock(POC). Peerspace/
    스페이스클라우드/아워플레이스 어디에도 없는 한국 시장 fit 차별점.

### 4.5 호스트 도구 차별점

16. **영업 외 시간 수익 시뮬레이터** _(구현됨, 사이클22)_ — 호스트 랜딩의 핵심 후크.
    "임대료는 24시간 나가는데 매출은 영업시간에만" 이라는 데드 캐피털 문제를, 업종·주당
    개방 시간·시간당 단가·예상 점유율 슬라이더로 조절하면 **월 추가 순수입**(수수료 12%
    제외) + 연 환산액을 실시간 계산해 보여준다. "영업 외 시간은 추가 비용 거의 없이
    순수익" 이라는 프레이밍으로 부업형 호스트의 등록 동기를 자극. 카페 기준 주 16h·점유
    55% → 월 +266만원 같은 구체 숫자가 즉시 CTA("내 가게로 월 266만원 더 벌기")로 연결.

17. **시장가 제안기(Smart Pricing)** _(구현됨, 사이클7)_ — 공간 등록 가격 입력 옆에
    같은 지역·카테고리·인원 비교 가격 분포(p25 / median / p75)와 최근 30일 슬롯 점유율을
    합쳐 추천 시간당 가격을 제시. 점유율 0.5 기준 ±20%p 보정. Airbnb Smart Pricing 의
    마켓플레이스 버전으로, 한국 공간대여 어디에도 없는 부업 호스트용 가격 결정 도구.
    "내 가격 +31%" 같은 차이 뱃지로 즉시 비교 → 한 번에 적용 가능.

18. **외부 일정 차단 + iCal 동기화** _(구현됨, 사이클12)_ — 호스트의 가장 큰 마찰점인
    **더블 부킹 공포**를 해소. 카톡으로 받은 외부 예약, 본인 일정은 수동 차단으로 등록하고,
    구글·네이버 캘린더는 iCal URL 한 줄 입력으로 정기 동기화. 외부 이벤트 시간대와
    겹치는 슬롯은 검색/예약에서 자동 제외돼 더블 부킹 가능성 차단. 동기화 실패는
    `lastError`로 노출. Airbnb/Peerspace 의 핵심 호스트 기능, 한국 공간대여 전무.

19. **호스트 답글 시스템 + 답글률 뱃지** _(구현됨, 사이클10)_ — 게스트 후기에 호스트가
    답글을 달 수 있고, "후기 답글률 100% (28건)"이 디테일 페이지의 호스트 카드에 자동
    뱃지로 노출. answered/total 캐시는 답글 작성 시 즉시 재계산. `/host/reviews` 페이지는
    전체·미답글·답글 완료 탭으로 분리되어 호스트가 빠르게 답할 수 있고, 답글 등록 후
    공간 디테일에 들여쓰기 박스로 노출된다. Airbnb Superhost 의 핵심 신뢰 메트릭이지만
    스페이스클라우드/아워플레이스 어디에도 없다. 단순 별점이 아니라 "호스트가 후기에
    답하는가"가 게스트 결제 전환의 강력한 시그널.

20. **법인 결제 + 세금계산서 (Corporate Mode)** _(구현됨, 사이클18)_ — Use Case Discovery
    의 사내 워크샵·팀빌딩 use case 와 직접 연결되는 B2B 매출 견인. 한국 B2B 결제 마찰점인
    세금계산서 발행을 한 번 등록(`/me/corporate`)해두고 예약 시 토글로 첨부 → 회사 정보
    스냅샷이 예약에 동결, 호스트의 예약 디테일에 "법인 결제 · 세금계산서" 카드 자동 노출
    (회사명·사업자번호·대표자명·세금계산서 수신 이메일·과세 유형). 한국 공간대여(스페이스
    클라우드/아워플레이스)는 모두 "관리자에게 직접 문의" 처리. 사업자번호 정규식 검증.

21. **자연어 검색 (Smart Search)** _(구현됨, 사이클19)_ — "강남 야간 와인 30명 주차" 같이
    한국어 자연어 입력을 정형 필터(region/category/capacity/useCases/amenities/q)로 자동
    파싱. 사전 기반 substring 매칭(LLM 없이): 행정구역·지역 별칭(강남/홍대/연남/판교) ·
    카테고리 키워드 12종 · "30명/8인/people" 인원 정규식 · use case 12종 · amenity 18종 ·
    잔여 키워드는 q 폴백. HeroSearch 에 "자유롭게 검색"/"상세 검색" 토글 + 실시간 chip
    미리보기(📍 서울 · 🍻 바·호프 · 👥 30명 · ✅ 주차). 한국 공간대여(스페이스
    클라우드/아워플레이스)는 모두 정형 dropdown 필터만 제공.

22. **개인화 추천 (Recently Viewed + For You)** _(구현됨, 사이클17)_ — localStorage 의
    `recently-viewed` 슬러그 큐(최대 12개, FIFO·중복제거)를 기반으로 두 표면 추가:
    `/me` 페이지의 "최근 본 공간" 그리드(둘러본 순서대로), 홈의 "당신이 본 공간과 비슷한
    무드" 캐러셀(시드 ≥3개 일 때 활성). 추천 점수 = 카테고리 매칭(+2) × use-case 매칭(+1)
    × 평점 보너스(0~1). Airbnb/Booking.com 표준이지만 한국 공간대여(스페이스클라우드/
    아워플레이스/스카이뷰)는 정적 카테고리만 노출. 가입 없이도 작동.

23. **도착 가이드 (Arrival Kit)** _(구현됨, 사이클16)_ — 호스트가 venue별로 한 번
    입력해두면(주차·출입문 코드·Wi-Fi SSID/PW·분리수거·비상연락처·기타) 결제 완료
    게스트의 예약 디테일에 자동 노출. 비밀번호는 기본 마스킹+눈 토글, 코드·연락처·주소는
    한 클릭 클립보드 복사. 호스트의 카톡 반복 응대 시간 절감 + 게스트의 당일 헤매기 해소.
    한국 공간대여 어디에도 체계화 안 됨(즉흥 카톡 안내).

24. **호스트 수요 히트맵 (Demand Heatmap)** _(구현됨, 사이클15)_ — 호스트 대시보드에
    7×24 그리드 위젯. 최근 60일 PAID/COMPLETED 예약 시각을 요일·시간 셀에 누적, 점유율
    기반 색 강도로 시각화. 상단 top 3 슬롯("#1 수요일 20시 점유율 84% · 12건 예약")
    카드로 가장 인기 있는 시간대 직접 노출. "인기 시간대는 동적 가격으로 인상하거나
    슬롯을 추가하세요" CTA 로 호스트의 다음 행동 유도. 한국 공간대여에 없는
    호스트 인사이트 차별점.

25. **활동 시그널 (Activity Pulse)** _(구현됨, 사이클14)_ — 카드와 디테일에 동적
    시그널: 평점 ≥4.85 + 후기 ≥10건이면 "🔥 인기", 등록 7일 이내면 "🆕 신규". 디테일
    페이지의 "이 공간의 활동" 카드는 최근 30일 PAID/COMPLETED 예약 건수 + 누적 조회수를
    big-number 로 노출. Airbnb/Booking.com 의 표준 FOMO·신뢰 트리거이지만 한국
    공간대여(스페이스클라우드/아워플레이스) 모두 정적 정보만 제공.

26. **빈자리 알림 (Slot Waitlist)** _(구현됨, 사이클23)_ — 인기 공간이 꽉 찼을 때 게스트가
    "빈자리 알림"을 신청, 예약 취소·환불이 발생하면 대기자 전원에게 즉시 알림(중복 방지
    `notifiedAt`). 디테일 예약 패널에 "지금 N명이 빈자리를 기다려요" 사회적 증거가 떠
    예약 압력을 만들고, 호스트에겐 대기 수요가 영업 외 시간을 더 열 동기(= fill-rate↑ =
    추가 수입)가 된다. 바운스되던 수요를 붙잡는 마켓플레이스 유동성 레버 — 한국
    공간대여 전무.

### 4.6 무드 차별점

26. **"조용한 럭셔리" 큐레이션** — 정보 나열식 카드 대신 풀블리드 무드 사진 + 동선 영상
    30초. SNS 공유 최적화.

## 6. 핵심 사용자 플로우

### 6.1 게스트 — 검색에서 리뷰까지

```
홈 → 용도×시간대×인원 검색 → 결과 카드 그리드/지도 → 공간 상세
  → 날짜/시간 선택 → 인원·옵션 입력 → 결제 → 예약 확정
  → (이용 당일) 체크인 코드 → 체크아웃 → 리뷰 작성
```

### 6.2 호스트 — 등록에서 정산까지

```
호스트 가입 → 사업자등록증 OCR 인증 → 영업시간 등록
  → 공간 정보(사진·평수·룰) → 가격·청소 윈도우 설정
  → 자동 슬롯 캘린더 검토 → 게시
  → (예약 발생) 알림 → 승인/즉시예약 → 이용 → 정산(7/14일)
```

### 6.3 예약 상태 머신

```
REQUESTED → APPROVED → PAID → CHECKED_IN → CHECKED_OUT → COMPLETED → REVIEWED
                   ↓        ↓
                 CANCELED_BY_HOST / CANCELED_BY_GUEST → REFUNDED
```

## 8. 정책 요약

- **취소·환불**: 이용일 7일 전 100%, 3일 전 50%, 1일 전 20%, 당일 0%. 호스트 귀책은 100%.
- **수수료**: 호스트 측 12%, 게스트 측 무료(가격에 포함). 결제 PG 비용은 플랫폼 부담.
- **정산**: 이용 완료 +7일(부업 호스트) / +14일(사업자 호스트, 세금계산서).
- **보증금**: 옵션. 카드 사전 인증 + 분쟁 없으면 7일 내 자동 해제.
- **신원 확인**: 게스트는 본인 인증(휴대폰), 호스트는 사업자등록증 인증.

## 9. 비기능 요구사항

- **성능**: LCP ≤ 2.5s (모바일 3G Fast), 검색 응답 ≤ 500ms (p95).
- **가용성**: 99.5% (Phase 1), 99.9% (Phase 2).
- **보안**: OWASP Top 10 대응, JWT + Refresh Token Rotation, argon2id 해싱.
- **접근성**: WCAG 2.1 AA, 키보드 내비게이션, prefers-reduced-motion 대응.
- **국제화**: 기본 ko, 구조는 i18n-ready.
- **반응형**: 모바일 퍼스트 (320~430px), 태블릿 (768px), 데스크탑 (1024~1920px).

## 10. 관리자(Admin) 기능

### 10.1 권한 모델

```
superadmin  → 전 권한 (역할 변경, 시스템 설정, 감사 로그 영구 삭제)
admin       → 사용자/공간/예약/리뷰 모더레이션, 통계, 알림 발송
host        → 본인 공간/예약/정산 관리 (호스트 대시보드)
user        → 게스트 기본 권한
```

### 10.3 대시보드 KPI 카드 (실시간)

### 10.5 모더레이션 SLA

- 위반 확정 → 조치 ≤ 48h
- 분쟁 발생 → 1차 중재 ≤ 72h

## 11. 비공식 정책 / 윤리

- 결혼·돌잔치 등 비주류 행사 우대 광고 금지 (모든 모임을 동등 취급).
- 위치 정보 정확도는 동/번지 수준, 정확한 호수는 결제 완료 후 공개.
- 호스트·게스트 리뷰는 더블블라인드 — 양측 모두 작성 또는 14일 경과 후 공개.
- 분쟁 시 플랫폼이 1차 중재, 보험·법률 사항은 파트너 연결.
